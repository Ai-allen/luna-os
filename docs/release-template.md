# LunaOS Release Template

## Scope

This document defines the reusable release directory and publication template
for frozen release candidates and future GA releases.

## Version Stem Rule

Every published release must use a single version stem across:

- directory name
- image file names
- release notes
- manifest file
- checksum file

Examples:

- release candidate:
  - `rc3-frozen-2026-04-10`
- future GA:
  - `ga-YYYY-MM-DD`

## Release Directory Template

```text
release/
  <version-stem>/
    image/
    notes/
    checksums/
    logs/
    <version-stem>-manifest.txt
```

## Required Image Rules

- all published image names must begin with the same version stem
- each image artifact must be generated by the release build being published
- image variants such as disk, esp, installer, or bootloader should append a
  suffix after the stem rather than replacing it

## Required Notes Rules

- notes directory must include release notes
- notes directory must include operations guidance
- notes directory must include the release gate
- notes directory may include packaging rules and roadmap material

## Required Checksum Rules

- checksum algorithm must be `sha256`
- checksum file name must include the version stem
- every published image artifact must have a checksum entry
- checksum entries must use the published relative artifact path

## Required Manifest Rules

The manifest must record:

- version stem
- release date
- image artifact list with file size
- note files included in the package
- archived log files included in the package

## Required Log Rules

- logs must include the automated release gate outputs
- logs must include error logs when the automated gate produces them
- logs must include regression summaries when those summaries are generated
- logs must be stored under the published release directory rather than only in
  the working tree
